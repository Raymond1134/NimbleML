## Troubleshooting CUDA / NaN
  - Use batch_size=8 in gpt_toy_config.toml (not 10+ on 8 GB VRAM).
  - grad_accum_steps simulates a larger batch without extra VRAM (e.g. 8 x 32 = effective 256).
  - gc_every=1 reclaims CuPy pool memory each step; raise it only if steps get slower.
  - After any CUDA crash, start a fresh terminal/Python process (GPU context is poisoned).
  - Starting loss should be ~9–10, not ~300 (embedding init is 0.02 std).
  - Training skips steps with non-finite loss/grad and logs a warning.
